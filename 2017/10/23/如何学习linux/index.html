如何学习Linux

  1. 动手实验
    关于Linux网上有很多教程，还有大量的书籍可供参考。但是请记住，未经自己亲手实践的教程就不属于自己。由于种种例如工程师‘留一手’的考虑，教程上往往只提及怎样做是对的而不提及可能遇到的问题。例如某些最小化安装Linux导致的相关库模块的缺失，若不实验教程上的问题就无法暴露出来。

  2. 不怕出错
    例如配置一个服务，工程师之间的差别不在于谁会配置，而在于出现问题后谁能准确判断并解决。这将直接关系到你的收入。今天碰到的问题分析、解决、总结后将直接变成你工作中的财富。

  3. 做好笔记
    经过自己归纳总结的笔记就是生产工具。工作中直接复制粘贴笔记中的代码，要求迅速准确。

    • 建议使用云笔记
      为知笔记(推荐使用)
      Microsoft Onenote
      有道云笔记
      Evernote
    • 使用Markdown语言记录笔记
      最流行的轻量级标记语言，生成的文档可转换成 html pdf

  4. 归纳总结
    吾生也有涯，学也无涯，以有涯随无涯殆已。
    工作中遇到的问题不可能在实验中都遇到过，学习技术重要的是归纳和总结。碰到的问题可能没遇到过，但知道以何种流程解决，解决问题的方法是学习技术的关键。对于linux来说最重要的就是日志。每解决一个问题请及时补充到你的笔记中。

  5. 英语
    linux中命令、日志都是英文，如果英文很差，请务必安装翻译软件辅助你学习。
